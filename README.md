## Overview

`financial-classification` is a machine learning final project home base for classifying financial data (transactions, expense types, and merchant categories).

## Project Structure

```text
financial-classification/
├── README.md
├── requirements.txt
├── data/                 # Raw and processed datasets
│   └── processed/        # Generated by data_preprocessing.py
├── src/                  # Core source code
│   ├── __init__.py
│   ├── data_preprocessing.py  # Shared cleaning, encoding, and splits
│   ├── featurization.py       # TF-IDF + OHE + StandardScaler transformer
│   ├── model_data.py          # Loads splits, fits transformer, returns arrays
│   ├── evaluate.py            # Shared metrics and confusion matrix
│   └── random_forest.py       # Random Forest model (Bradley)
└── models/               # Saved model artifacts (.joblib + confusion matrix PNGs)
```

## Quick Start

### 1) Clone the repository

```bash
git clone https://github.com/darielgu/financial-classification.git
cd financial-classification
```

### 2) Create a virtual environment

```bash
python -m venv .venv
source .venv/bin/activate
```

Windows (PowerShell):

```powershell
.venv\Scripts\Activate.ps1
```

### 3) Install dependencies

```bash
pip install -r requirements.txt
```

### 4) Run a model

```bash
# Random Forest
python -m src.random_forest
```

## Data Preprocessing

Run one shared pipeline to combine both datasets and generate reusable splits for all models:

```bash
python src/data_preprocessing.py
```

This creates:
- `data/processed/transactions_preprocessed.csv`
- `data/processed/train.csv`
- `data/processed/val.csv`
- `data/processed/test.csv`
- `data/processed/feature_columns.txt`

Preprocessing steps applied:
- Missing values dropped for required fields (`amount`, `date`); optional fields filled with `"unknown"`
- Duplicate records removed
- Category labels normalized and consolidated (e.g. Salary + Paycheck → `Income`; Restaurants + Fast Food + Food & Dining → `Dining`; Movies & DVDs + Television + Music → `Entertainment`) — 26 raw categories reduced to 21
- Text descriptions cleaned (lowercase, punctuation stripped)
- Date features extracted (`date_month`, `date_day_of_week`)
- Stratified 64 / 16 / 20 train / val / test split (`random_state=42`)

Use the same data loading interface across all models:

```python
from src.model_data import get_data_for_model

# Returns fitted numpy arrays — transformer is already fitted on x_train only
x_train, y_train, x_val, y_val, x_test, y_test, feature_transformer = get_data_for_model()
```

> **Note:** `x_train`, `x_val`, and `x_test` are dense numpy arrays ready to pass directly
> to any scikit-learn estimator. `feature_transformer` is a fitted `ColumnTransformer`
> — use `feature_transformer.transform(new_data)` at inference time.
